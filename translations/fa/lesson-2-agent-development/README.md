# درس ۲ توسعه عامل

به درس دوم دوره «ساخت عامل هوش مصنوعی از صفر تا تولید» خوش آمدید!

در این درس موارد زیر را پوشش خواهیم داد:

- ابزارهای ایجاد عامل‌های هوش مصنوعی ما
  
- دستورالعمل‌های راه‌اندازی منابع توسعه

- بهترین شیوه‌های توسعه عامل هوش مصنوعی
  
- بررسی کد برای ایجاد عامل‌های هوش مصنوعی ما
  
بیایید با نگاه کردن به ابزارهایی که برای ایجاد عامل‌های هوش مصنوعی استفاده می‌کنیم شروع کنیم.

## ابزارها و دستورالعمل‌های راه‌اندازی

### Microsoft Foundry

برای دسترسی به مدل‌های زبان بزرگ (LLMs) ما از [Microsoft Foundry](https://azure.microsoft.com/products/ai-foundry) استفاده خواهیم کرد. هزینه‌هایی برای استفاده از Foundry وجود دارد، بنابراین لطفاً اگر قبلاً دسترسی ندارید، حتماً دستورالعمل‌های راه‌اندازی حساب را دنبال کنید.

### مدل‌های OpenAI

نمونه‌های کد عامل در این دوره به گونه‌ای تنظیم شده‌اند که از مدل‌های OpenAI از طریق [Microsoft Foundry](https://azure.microsoft.com/products/ai-foundry) استفاده کنند.

برای یادگیری نحوه استقرار یک مدل با استفاده از Foundry از این راهنما استفاده کنید: [Deploy Microsoft Foundry Models in the Foundry portal](https://learn.microsoft.com/azure/ai-foundry/foundry-models/how-to/deploy-foundry-models?view=foundry-classic)

برای این دوره یکی از مدل‌های GPT-4.1 یا نسخه‌های بعدی را انتخاب کنید.

### چارچوب عامل مایکروسافت

همانطور که قبلاً اشاره شد، ما از [چارچوب عامل مایکروسافت](https://github.com/microsoft/agent-framework) برای ساخت و هماهنگی عامل‌های هوش مصنوعی خود استفاده خواهیم کرد.

برای نصب چارچوب عامل مایکروسافت و بسته‌های مورد نیاز دیگر، دستور زیر را در دایرکتوری ریشه این پروژه اجرا کنید:

```bash
pip install -r requirements.txt
```

### تنظیم متغیرهای .env

برای اجرای نمونه‌های کد در این دوره، باید یک فایل `.env` در دایرکتوری ریشه این پروژه بسازید.

برای سهولت، می‌توانید فایل `.env.example` ارائه‌شده را کپی کنید:

```bash
cp .env.example .env
```
