# Lesson 2 Agent Development

Добро пожаловать на второй урок курса «Создание AI-агентов от нуля до производства»!

В этом уроке мы рассмотрим:

- Инструменты для создания наших AI-агентов
  
- Инструкции по настройке наших ресурсов для разработки

- Лучшие практики разработки AI-агентов
  
- Поэтапный разбор кода для создания наших AI-агентов
  
Давайте начнем с обзора инструментов, которые мы будем использовать для создания наших AI-агентов.

## Инструменты и инструкции по настройке

### Microsoft Foundry

Для доступа к крупным языковым моделям (LLM) мы будем использовать [Microsoft Foundry](https://azure.microsoft.com/products/ai-foundry). Использование Foundry связано с затратами, поэтому пожалуйста, следуйте инструкциям по настройке аккаунта, если у вас еще нет доступа.

### OpenAI модели

Примеры кода агентов в этом курсе настроены на использование моделей OpenAI через [Microsoft Foundry](https://azure.microsoft.com/products/ai-foundry).

Используйте это руководство, чтобы узнать, как развернуть модель с помощью Foundry: [Deploy Microsoft Foundry Models in the Foundry portal](https://learn.microsoft.com/azure/ai-foundry/foundry-models/how-to/deploy-foundry-models?view=foundry-classic)

Выберите одну модель GPT-4.1 или более позднюю для этого курса.

### Microsoft Agent Framework

Как упоминалось ранее, мы будем использовать [Microsoft Agent Framework](https://github.com/microsoft/agent-framework) как для создания, так и для оркестрации наших AI-агентов.

Чтобы установить Microsoft Agent Framework и другие необходимые пакеты, выполните следующую команду в корневой директории этого проекта:

```bash
pip install -r requirements.txt
```

### Настройка переменных .env

Для запуска примеров кода из этого курса вам нужно создать файл `.env` в корневой директории проекта.

Для удобства вы можете скопировать предоставленный файл `.env.example`:

```bash
cp .env.example .env
```
