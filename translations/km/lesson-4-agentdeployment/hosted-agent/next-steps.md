## ជំហានបន្ទាប់

### ផ្តល់សេវារចនាសម្ព័ន្ធ និងដាក់បញ្ចូលកូដកម្មវិធី

រត់ `azd up` ដើម្បីផ្តល់សេវារចនាសម្ព័ន្ធរបស់អ្នក និងដាក់បញ្ចូលទៅ Azure ក្នុងជំហានតែមួយ (ឬរត់ `azd provision` រួច `azd deploy` ដើម្បីបញ្ចប់ភារកិច្ចជាប្រចាំ)។ ចូលចិត្តទៅកាន់ចំណុចសេវាកម្មដែលបានរាយបញ្ជីដើម្បីមើលកម្មវិធីរបស់អ្នកដំណើរការឡើងវិញ!

ដើម្បីដោះស្រាយបញ្ហាណាមួយ សូមមើល [ដោះស្រាយបញ្ហា](#ដោះស្រាយបញ្ហា)។

### កែប្រែរចនាសម្ព័ន្ធ

ដើម្បីពិពណ៌នារចនាសម្ព័ន្ធ និងកម្មវិធី `azure.yaml` ត្រូវបានបន្ថែម។ ឯកសារនេះមានសេវា និងធនធានទាំងអស់ដែលពិពណ៌នាអំពីកម្មវិធីរបស់អ្នក។

ដើម្បីបន្ថែមសេវាឬធនធានថ្មីៗ សូមរត់ `azd add`។ អ្នកអាចកែសម្រួលផ្ទាល់ឯកសារ `azure.yaml` បើមានត្រូវការ។

### ការត្រៀមទៅរកសមត្ថភាពផលិត

ពេលត្រូវការ, `azd` បង្កើតរចនាសម្ព័ន្ធដែលត្រូវការ ជាកូដក្នុងចងចាំ ហើយប្រើវា។
ប្រសិនបើអ្នកចង់មើល ឬកែប្រែរចនាសម្ព័ន្ធដែល `azd` ប្រើ សូមរត់ `azd infra gen` ដើម្បីរក្សាទុកវាទៅលើថាស។

បើអ្នកធ្វើបែបនេះ, តម្រូវឲ្យបង្កើតថតបន្ថែមខាងក្រោម៖

```yaml
- infra/            # Infrastructure as Code (bicep) files
  - main.bicep      # main deployment module
  - resources.bicep # resources shared across your application's services
  - modules/        # Library modules
```

*ចំណាំ*: ពេលដែលអ្នកបានបង្កើតរចនាសម្ព័ន្ធរបស់អ្នកទៅលើថាសហើយ, ឯកសារទាំងនោះគឺជាប្រភពពិតសម្រាប់ azd។ ការផ្លាស់ប្តូរណាមួយនៅក្នុង `azure.yaml` (ដូចជា ការដឹកនាំតាម `azd add`) នឹងមិនបង្ហាញក្នុងរចនាសម្ព័ន្ធរហូតដល់អ្នកបង្កើតវាឡើងវិញជាមួយ `azd infra gen` ម្តងទៀតទេ។ វានឹងរំលឹកអ្នកមុនពេលគេលឯកសារ។ អ្នកអាចប្រើ `--force` ដើម្បីបង្ខំឲ្យ `azd infra gen` គេលឯកសារដោយមិនរំលឹក។

ចុងក្រោយ, រត់ `azd pipeline config` ដើម្បីកំណត់បំពង់ចេញ CI/CD។

## វិក័យប័ត្រ

ចូលទៅកាន់ទំព័រ *Cost Management + Billing* នៅក្នុង Azure Portal ដើម្បីតាមដានការចំណាយបច្ចុប្បន្ន។ សម្រាប់ព័ត៌មានលម្អិតអំពីរបៀបដែលអ្នកត្រូវបានគិតថ្លៃ និងរបៀបដែលអ្នកអាចតាមដានការចំណាយក្នុងការជាវ Azure របស់អ្នក សូមទៅកាន់ [billing overview](https://learn.microsoft.com/azure/developer/intro/azure-developer-billing)។

## ដោះស្រាយបញ្ហា

Q: ខ្ញុំបានចូលទៅកាន់ចំណុចសេវាកម្មដែលបានរាយបញ្ជី ហើយខ្ញុំឃើញទំព័រពណ៌ស ទំព័រស្វាគមន៍ទូទៅ ឬទំព័របង្ហាញកំហុស។

A: សេវាកម្មរបស់អ្នកអាចមានបញ្ហាក្នុងការចាប់ផ្តើម ឬវាអាចខ្វះការកំណត់ថា។ ដើម្បីសម្លឹងពិចារណាបន្ថែម៖

1. រត់ `azd show`។ ចុចលើតំណភ្ជាប់ដែលនៅក្រោម "View in Azure Portal" ដើម្បីបើកក្រុមធនធានក្នុង Azure Portal។
2. រុករកទៅកាន់សេវាកម្ម Container App ដែលពុំអាចដាក់បញ្ចូលបាន។
3. ចុចលើកំណែដែលបរាជ័យនៅក្រោម "Revisions with Issues"។
4. សូមពិនិត្យ "Status details" ដើម្បីទទួលបានព័ត៌មានបន្ថែមអំពីប្រភេទបរាជ័យ។
5. សង្កេតចេញ log ពី Console log stream និង System log stream ដើម្បីស្វែងរកកំហុសណាមួយ។
6. ប្រសិនបើ log ត្រូវបានសរសេរទៅលើថាស សូមប្រើ *Console* ក្នុងការរុករក ដើម្បីភ្ជាប់ទៅចូល shell នៅក្នុង container ដែលកំពុងដំណើរការ។

សម្រាប់ព័ត៌មានបន្ថែមអំពីការដោះស្រាយបញ្ហា សូមទៅកាន់ [Container Apps troubleshooting](https://learn.microsoft.com/azure/container-apps/troubleshooting)។

### ព័ត៌មានបន្ថែម

សម្រាប់ព័ត៌មានបន្ថែមអំពីការកំណត់គម្រោង `azd` របស់អ្នក ជុំវិញ សូមចូលទៅកាន់ឯកសារផ្លូវការរបស់យើង [docs](https://learn.microsoft.com/azure/developer/azure-developer-cli/make-azd-compatible?pivots=azd-convert)។
