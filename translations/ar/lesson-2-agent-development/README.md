# الدرس 2 تطوير الوكيل

مرحبًا بكم في الدرس الثاني من دورة "بناء وكيل ذكاء اصطناعي من الصفر إلى الإنتاج"!

في هذا الدرس سنغطي:

- الأدوات لإنشاء وكلاء الذكاء الاصطناعي لدينا
  
- تعليمات إعداد موارد التطوير لدينا

- أفضل الممارسات في تطوير وكلاء الذكاء الاصطناعي
  
- جولة في الكود لإنشاء وكلاء الذكاء الاصطناعي لدينا
  
لنبدأ بالنظر إلى الأدوات التي سنستخدمها لإنشاء وكلاء الذكاء الاصطناعي لدينا.

## الأدوات وتعليمات الإعداد

### Microsoft Foundry

للوصول إلى نماذج اللغة الكبيرة (LLMs) سنستخدم [Microsoft Foundry](https://azure.microsoft.com/products/ai-foundry). توجد تكاليف مرتبطة باستخدام Foundry لذا يرجى التأكد من اتباع تعليمات إعداد الحساب إذا لم يكن لديك وصول بالفعل.

### نماذج OpenAI

نماذج الكود الخاصة بالوكيل في هذه الدورة معدة لاستخدام نماذج OpenAI من خلال [Microsoft Foundry](https://azure.microsoft.com/products/ai-foundry).

استخدم هذا الدليل لتعلم كيفية نشر نموذج باستخدام Foundry: [نشر نماذج Microsoft Foundry في بوابة Foundry](https://learn.microsoft.com/azure/ai-foundry/foundry-models/how-to/deploy-foundry-models?view=foundry-classic)

اختر نموذج GPT-4.1 أو أحدث لهذه الدورة.

### إطار عمل Microsoft Agent

كما ذكر سابقًا، سنستخدم [إطار عمل Microsoft Agent](https://github.com/microsoft/agent-framework) لكل من إنشاء وتنسيق وكلائنا الذكاء الاصطناعي.

لتثبيت إطار عمل Microsoft Agent والحزم المطلوبة الأخرى، شغّل الأمر التالي أثناء وجودك في الدليل الجذري لهذا المشروع:

```bash
pip install -r requirements.txt
```

### إعداد متغيرات .env

لتشغيل عينات الكود في هذه الدورة، ستحتاج إلى إنشاء ملف `.env` في الدليل الجذري لهذا المشروع.

لتسهيل الأمر، يمكنك نسخ ملف `.env.example` المقدم:

```bash
cp .env.example .env
```
