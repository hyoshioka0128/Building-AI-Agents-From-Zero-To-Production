# Lesson 2 Agent Development

Ласкаво просимо до другого уроку курсу "Створення AI агента від нуля до виробництва"!

У цьому уроці ми розглянемо:

- Інструменти для створення наших AI агентів
  
- Інструкції з налаштування наших ресурсів для розробки

- Кращі практики розробки AI агентів
  
- Покроковий розбір коду для створення наших AI агентів
  
Почнемо з огляду інструментів, які ми будемо використовувати для створення наших AI агентів.

## Інструменти та інструкції з налаштування

### Microsoft Foundry

Для доступу до великих мовних моделей (LLM) ми будемо використовувати [Microsoft Foundry](https://azure.microsoft.com/products/ai-foundry). Використання Foundry пов’язане з витратами, тому, будь ласка, дотримуйтесь інструкцій для налаштування облікового запису, якщо у вас ще немає доступу.

### Моделі OpenAI

Приклади коду агента в цьому курсі налаштовані на використання моделей OpenAI через [Microsoft Foundry](https://azure.microsoft.com/products/ai-foundry).

Використовуйте цей посібник, щоб дізнатись, як розгорнути модель за допомогою Foundry: [Deploy Microsoft Foundry Models in the Foundry portal](https://learn.microsoft.com/azure/ai-foundry/foundry-models/how-to/deploy-foundry-models?view=foundry-classic)

Виберіть модель GPT-4.1 або пізнішої версії для цього курсу.

### Microsoft Agent Framework

Як згадувалось раніше, ми будемо використовувати [Microsoft Agent Framework](https://github.com/microsoft/agent-framework) для створення та оркестрації наших AI агентів.

Щоб встановити Microsoft Agent Framework та інші необхідні пакети, виконайте наступну команду у кореневій директорії цього проєкту:

```bash
pip install -r requirements.txt
```

### Налаштування змінних .env

Для запуску прикладів коду в цьому курсі вам потрібно створити файл `.env` у кореневій директорії цього проєкту.

Щоб полегшити це, ви можете скопіювати наданий файл `.env.example`:

```bash
cp .env.example .env
```
